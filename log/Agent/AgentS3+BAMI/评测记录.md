## 三、实验过程说明

### API 余额耗尽问题

本次实验在运行后期（2026-06-01 下午）遭遇 dmxapi.cn API key 余额耗尽：

```
Token amount has been exhausted
token.RemainQuota = -282135
```

受影响症状：GPT-4o 调用失败 → 规划为空 → agent 每步只执行 `time.sleep(1.333)` 直至达到步数上限。

受影响任务已从结果目录中删除（共 26 个），不计入统计：
- thunderbird：8 个（几乎全部）
- multi_apps：16 个
- gimp：1 个
- os：1 个

vlc（17 个）和 vs_code（23 个）完全未能开始评估，需补跑。

### 实验配置

配置文件：[pyautogui/screenshot/gpt-4o/args.json](pyautogui/screenshot/gpt-4o/args.json)

| 参数 | 值 |
|------|----|
| provider_name | docker |
| model（规划） | gpt-4o |
| model_url | https://www.dmxapi.cn/v1 |
| BAMI server | http://127.0.0.1:8001（Holo2-8B + 裁剪判别模型，GPU 4）|
| ground_model（fallback） | ByteDance-Seed/UI-TARS-1.5-7B |
| num_envs | 8 |
| max_steps | 50 |
| sleep_after_execution | 3.0s |

**架构：**

```
AgentS3 worker(s)
    ↓ 规划
  gpt-4o API
  (dmxapi.cn)
    ↓ 截图 + 坐标请求
  BAMI server (port 8001, GPU 4)        → Holo2-8B 初步定位
    ├── 裁剪判别模型 (Qwen3-VL-8B)      → 验证/精炼坐标
    └── fallback: UI-TARS vllm (port 8000, GPU 6/7)
    ↓ 执行
  Docker 容器 (Ubuntu VM)
```
